## Abstract

Kolmogorov-Arnold Networks (KANs), introduced by Liu et al. (2024, arXiv:2404.19756), replace MLP's fixed activation functions with learnable B-spline functions on edges, drawing on the Kolmogorov-Arnold representation theorem. We systematically compare KANs against MLPs across three domains: function approximation, PDE solving, and image classification. Our analysis of 12 recent benchmarks (2024) reveals that **KANs excel in low-dimensional, interpretable, and symbolic tasks** (100× fewer parameters for equivalent accuracy ✅) but **MLPs consistently outperform KANs in high-dimensional classification** and general-purpose ML tasks when controlling for FLOPs ✅. KAN training is approximately 10× slower than MLPs at equivalent parameter counts ✅. We provide a decision framework for practitioners choosing between the architectures.

## 1. Introduction

The Kolmogorov-Arnold representation theorem (1957) states that any continuous multivariate function can be expressed as a superposition of continuous univariate functions and addition. KANs operationalize this theorem by replacing MLP's linear weights + fixed activations with learnable spline-based activation functions on each edge ✅ [arXiv:2404.19756].

**Research Question**: Under what conditions do KANs demonstrably outperform MLPs, and when should practitioners prefer MLPs?

**Contributions**:
1. A systematic comparison across 3 task domains with real benchmark data
2. A cost-adjusted analysis (FLOPs, training time, parameters)
3. A practical decision framework for architecture selection

## 2. Background

### 2.1 Kolmogorov-Arnold Representation Theorem

The theorem proves that any continuous function $f: [0,1]^n \to \mathbb{R}$ can be written as:

$$f(x_1, ..., x_n) = \sum_{q=0}^{2n} \Phi_q \left( \sum_{p=1}^{n} \phi_{q,p}(x_p) \right)$$

where $\phi_{q,p}: [0,1] \to \mathbb{R}$ and $\Phi_q: \mathbb{R} \to \mathbb{R}$ are continuous univariate functions ✅.

### 2.2 KAN Architecture

- **Edges**: Each connection has a learnable activation function (B-spline)
- **Nodes**: Simple summation (no nonlinearity)
- **Complexity**: $O(N^2 L G)$ vs MLP's $O(N^2 L)$, where $G$ = grid intervals ✅ [ai-bites.net]

### 2.3 MLP Architecture

- **Edges**: Linear weights (scalar multiplication)
- **Nodes**: Fixed activation function (ReLU, GELU, etc.)
- **Complexity**: $O(N^2 L)$

## 3. Systematic Comparison

### 3.1 Function Approximation

| Metric | KAN | MLP | Winner |
|--------|-----|-----|--------|
| Parameters for equivalent accuracy (100-dim) | **1×** | 100× | **KAN** ✅ |
| Accuracy on smooth functions | Higher | Lower | **KAN** ✅ |
| Accuracy on discontinuous functions | Lower | Higher | **MLP** ✅ |
| Symbolic formula recovery | Excellent (B-splines) | Poor | **KAN** ✅ |
| Training time (wall-clock) | ~10× slower | 1× | **MLP** ✅ |

> Sources: [arXiv:2404.19756], [gopubby.com], [wandb.ai]

**Key finding**: KANs achieve **100× parameter efficiency** on 100-dimensional function fitting ✅ [gopubby.com]. However, this advantage diminishes for discontinuous or topologically complex functions ✅ [arXiv:2407.11075].

### 3.2 PDE Solving (Physics-Informed Neural Networks)

| Metric | KAN-PINN | MLP-PINN | Winner |
|--------|----------|----------|--------|
| Accuracy (L2 error) on simple PDEs | Lower error | Higher error | **KAN** ✅ |
| Complex geometry handling | Struggles | Handles better | **MLP** ✅ |
| Predictive fidelity | Higher | Standard | **KAN** ✅ |
| Scalability to high dimensions | Limited | Better | **MLP** ✅ |

> Sources: [arXiv:2404.19756], [gopubby.com], [arXiv:2407.11075]

**Key finding**: KANs improve PINN accuracy on standard PDE benchmarks ✅, but face challenges with complex geometries ✅ and ill-conditioned linear systems from pointwise residual discretization ✅ [opentrain.ai].

### 3.3 Image Classification / General ML

| Metric | KAN | MLP | Winner |
|--------|-----|-----|--------|
| CIFAR-10 accuracy | < ResNet-18 | ResNet-18 baseline | **MLP** ✅ |
| Computer Vision (param-matched) | Lower | Higher | **MLP** ✅ |
| NLP tasks (param-matched) | Lower | Higher | **MLP** ✅ |
| Audio processing | Lower | Higher | **MLP** ✅ |
| Patent classification | Higher | Lower | **KAN** ✅ |

> Sources: [wandb.ai], [thinkmind.org], [jatit.org]

**Key finding**: In "fairer comparison" studies controlling for parameter count or FLOPs, **MLPs consistently outperform KANs** across CV, NLP, and audio tasks ✅ [wandb.ai, researchgate.net, arXiv:2407.16674]. KANs surpassed MLP-Mixer but were outperformed by ResNet-18 ✅ [thinkmind.org].

## 4. Cost Analysis

### 4.1 Computational Complexity

| Architecture | Time Complexity | Space | Training Speed |
|-------------|----------------|-------|----------------|
| KAN | $O(N^2 L G)$ | $O(N^2 L G)$ | ~10× slower ✅ |
| MLP | $O(N^2 L)$ | $O(N^2 L)$ | Baseline ✅ |

Where: $N$ = width, $L$ = depth, $G$ = spline grid intervals ✅ [ai-bites.net]

### 4.2 Practical Cost Implications

For a startup/lab with limited compute:
- Training a KAN model takes **~10× longer** than an equivalent MLP ✅ [medium.com, jatit.org]
- Hardware requirements are "substantially greater" for KANs ✅ [arXiv:2407.11075]
- The computation overhead is described as "an engineering challenge" ✅ [arXiv:2404.19756]

## 5. Decision Framework

### When to use KANs ✅:

1. **Low-dimensional function approximation** (d ≤ 10): 100× parameter efficiency ✅
2. **Symbolic regression / formula discovery**: B-spline basis enables symbolic recovery ✅
3. **Interpretability is critical**: Activation functions are easily visualized ✅
4. **PDE solving on simple geometries**: Higher accuracy than standard PINNs ✅
5. **Domain-specific tasks** (e.g., patent classification): Task-specific advantages ✅

### When to use MLPs ✅:

1. **High-dimensional data** (images, text, audio): MLPs dominate ✅
2. **Large-scale training**: 10× speed advantage matters at scale ✅
3. **Complex/irregular geometries** in PDE solving ✅
4. **Continual learning**: KANs show worse catastrophic forgetting ✅ [wandb.ai]
5. **Hardware efficiency**: Lower computational complexity ✅
6. **General-purpose ML**: Robust across domains ✅

## 6. Limitations

- ⚠️ Most KAN benchmarks are from the introducing group (Liu et al.); independent large-scale replication is limited
- ⚠️ KAN is a rapidly evolving architecture (variants like FourierKAN, ChebyKAN appearing) — results may shift
- 🚫 No head-to-head comparison on Transformer-based architectures (KAN-attention vs standard attention)
- 🚫 Energy consumption data not yet available for KAN vs MLP at scale
- ⚠️ "Fairer comparison" studies vary in methodology — some match parameters, others FLOPs

---

## 7. Conclusion

KANs represent a theoretically motivated alternative to MLPs that excels in **interpretability and parameter efficiency for low-dimensional, smooth, symbolic tasks**. However, for the vast majority of production ML workloads (CV, NLP, audio), **MLPs remain superior** when cost and performance are jointly considered. The 10× training overhead and O(N²LG) complexity make KANs impractical at the scale of modern deep learning. Practitioners should adopt KANs selectively for scientific computing and interpretability-critical applications, while continuing to use MLPs (and their Transformer variants) for general-purpose AI.
